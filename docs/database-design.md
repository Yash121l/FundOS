# FundOS — Database Design

## Design Principles

1. **Append-only for critical data.** Founder updates, metrics, audit logs are never mutated. History is sacred.
2. **No schema drift.** All changes via Prisma migrations. No manual ALTER TABLE.
3. **Indexed foreign keys.** Every FK has an index. No full table scans on joins.
4. **Avoid N+1.** Schema designed to support efficient batch loading.
5. **Timestamps everywhere.** Every table has `createdAt` and `updatedAt`.

---

## Entity Relationship Summary

```
User ──────────────── Company (many-to-many via team membership)
Company ────────────── MetricSnapshot (1:many, append-only)
Company ────────────── FounderUpdate (1:many, append-only)
FounderUpdate ─────── Risk (1:many, generated by AI or manual)
FounderUpdate ─────── Opportunity (1:many, generated by AI)
FounderUpdate ─────── Action (1:many, suggested by AI or manual)
Company ────────────── Task (1:many)
LPReport ───────────── LPReportSection (1:many)
LPReport ───────────── Company (many-to-many via inclusion)
TrendFinding ──────── TrendEvidence → Company (many-to-many)
MarketSignal ──────── Company (many-to-many via relevance)
AuditLog ───────────── all entities (polymorphic)
```

### MetricSnapshot

Append-only. One record per reporting period per company. Never updated.

```prisma
model MetricSnapshot {
  id              String   @id @default(cuid())
  companyId       String
  period          String   // "2025-03" (YYYY-MM)
  
  // Revenue
  mrr             Float?   // Monthly Recurring Revenue (USD)
  arr             Float?   // Annual Run Rate (USD)
  revenueGrowthMom Float?  // MoM growth %
  revenueGrowthYoy Float?  // YoY growth %
  
  // Unit Economics
  grossMargin     Float?   // %
  nrr             Float?   // Net Revenue Retention %
  
  // Cash
  burnRate        Float?   // Monthly burn (USD)
  cashBalance     Float?   // Cash in bank (USD)
  runway          Float?   // Months of runway
  
  // Team
  headcount       Int?
  headcountChange Int?     // delta from previous period
  
  // Computed
  healthScore     Float?
  
  source          MetricSource @default(FOUNDER_UPDATE)
  createdAt       DateTime @default(now())

  company         Company @relation(fields: [companyId], references: [id])

  @@unique([companyId, period])
  @@index([companyId])
  @@index([period])
}

enum MetricSource {
  FOUNDER_UPDATE
  MANUAL
  IMPORT
}
```

### FounderUpdate

Append-only. Each submission is a new record.

```prisma
model FounderUpdate {
  id              String   @id @default(cuid())
  companyId       String
  period          String   // "2025-03"
  submittedById   String?  // Clerk user ID of submitter
  
  // Metrics (snapshot at time of submission)
  mrr             Float?
  burnRate        Float?
  cashBalance     Float?
  runway          Float?
  headcount       Int?
  fundraisingStatus FundraisingStatus @default(NOT_RAISING)
  fundraisingNote String?
  
  // Narrative (free text from founder)
  wins            String   // markdown
  risks           String   // markdown
  hiringNeeds     String?  // markdown
  additionalNotes String?
  
  // AI-generated outputs
  aiSummary       String?  // generated after submission
  aiProcessedAt   DateTime?
  
  // Review state
  reviewedAt      DateTime?
  reviewedById    String?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  company         Company      @relation(fields: [companyId], references: [id])
  risks           Risk[]
  opportunities   Opportunity[]
  actions         Action[]

  @@index([companyId])
  @@index([period])
  @@index([reviewedAt])
}

enum FundraisingStatus {
  NOT_RAISING
  EXPLORING
  ACTIVELY_RAISING
  TERM_SHEET
  CLOSED
}
```

### LPReport

```prisma
model LPReport {
  id           String         @id @default(cuid())
  title        String
  quarter      String         // "Q1 2025"
  status       ReportStatus   @default(GENERATING)
  generatedById String
  version      Int            @default(1)
  
  // Content
  markdownContent String?
  pdfUrl          String?
  
  // Metadata
  fundMetricsSnapshot Json?   // snapshot of fund metrics at generation time
  
  createdAt    DateTime       @default(now())
  updatedAt    DateTime       @updatedAt

  sections     LPReportSection[]
  companies    ReportCompany[]

  @@index([quarter])
  @@index([status])
}

enum ReportStatus {
  GENERATING
  READY
  EXPORTED
  ARCHIVED
}

model LPReportSection {
  id          String   @id @default(cuid())
  reportId    String
  title       String
  content     String   // markdown
  order       Int
  aiGenerated Boolean  @default(true)
  editedAt    DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  report      LPReport @relation(fields: [reportId], references: [id])

  @@index([reportId])
}

model ReportCompany {
  reportId    String
  companyId   String
  included    Boolean  @default(true)
  
  report      LPReport @relation(fields: [reportId], references: [id])
  company     Company  @relation(fields: [companyId], references: [id])

  @@id([reportId, companyId])
}
```

### TrendFinding

Cross-portfolio patterns detected by TrendDetectionAgent.

```prisma
model TrendFinding {
  id              String      @id @default(cuid())
  title           String      // "Enterprise sales slowdown"
  summary         String      // 1-2 sentence description
  category        TrendCategory
  severity        Severity    @default(MEDIUM)
  affectedCount   Int         // number of companies
  detectedAt      DateTime    @default(now())
  periodStart     DateTime    // analysis window start
  periodEnd       DateTime    // analysis window end
  status          TrendStatus @default(ACTIVE)
  dismissedAt     DateTime?
  actionId        String?     // linked action if one was created
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt

  evidence        TrendEvidence[]

  @@index([category])
  @@index([status])

  company         Company      @relation(fields: [companyId], references: [id])

  @@index([trendId])
  @@index([companyId])
}

enum TrendCategory {
  SHARED_RISK

  FUNDRAISING
  GROWTH_PATTERN
  MARKET_EVENT
  OPERATIONAL

enum TrendStatus {
```

### Metric Realism Rules

- Growth trajectories are non-linear. Include acceleration, plateaus, and declines.
- Burn rates scale with headcount. Series A companies burn $300K–800K/month.
- At-risk companies show deteriorating runway (< 6 months) or revenue decline.
- Watchlist companies show one warning signal — e.g., burn up but revenue growing.
- Healthy companies show stable-to-growing metrics with 12+ months runway.

## Query Patterns

### Dashboard load

```sql
-- Portfolio health summary (single query)
SELECT 
  health_status, 
  COUNT(*) as count
FROM companies 
WHERE status = 'ACTIVE'
GROUP BY health_status

-- Recent updates (last 7 days)
SELECT u.*, c.name, c.health_status
FROM founder_updates u
JOIN companies c ON u.company_id = c.id
WHERE u.created_at > NOW() - INTERVAL '7 days'
ORDER BY u.created_at DESC
LIMIT 10

-- Fund aggregates (from cached MetricSnapshots)
SELECT 
  SUM(mrr) as total_mrr,
  AVG(revenue_growth_mom) as avg_growth,
  SUM(burn_rate) as total_burn
FROM metric_snapshots
WHERE period = '2025-03'
```

### Company detail

```sql
-- Latest metrics + 12-month history
SELECT * FROM metric_snapshots
WHERE company_id = $1
ORDER BY period DESC
LIMIT 13

-- Open risks
SELECT * FROM risks
WHERE company_id = $1 AND status = 'OPEN'
ORDER BY severity DESC, created_at DESC

-- Recent updates with AI summary
SELECT * FROM founder_updates
WHERE company_id = $1
ORDER BY created_at DESC
LIMIT 10
```

### Trend detection input

```sql
-- All updates in last 90 days with narrative
SELECT u.*, c.name, c.sector
FROM founder_updates u
JOIN companies c ON u.company_id = c.id
WHERE u.created_at > NOW() - INTERVAL '90 days'
ORDER BY c.id, u.created_at DESC
```

## Indexing Strategy

Critical indexes beyond PKs and FKs:

```sql
-- Fast dashboard health summary
CREATE INDEX idx_companies_health_status ON companies(health_status) WHERE status = 'ACTIVE';

-- Fast updates inbox (unreviewed first)
CREATE INDEX idx_updates_reviewed_created ON founder_updates(reviewed_at, created_at DESC);

-- Fast metric history per company
CREATE INDEX idx_metrics_company_period ON metric_snapshots(company_id, period DESC);

-- Fast risk count per company
CREATE INDEX idx_risks_company_status ON risks(company_id, status);

-- Trend evidence lookup
CREATE INDEX idx_trend_evidence_trend ON trend_evidence(trend_id);
```
